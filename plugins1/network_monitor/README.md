# 网络监控内置插件

插件使用 Windows IP Helper API `GetIfTable2` 汇总活动非回环网卡的累计收发字节，并通过相邻采样差计算系统实时网速。v2rayN 的 Xray Core 通过 HTTP Metrics `/debug/vars` 提供代理累计流量。插件汇总 `stats.outbound` 中名称以 `proxy` 开头的出站。直连网速按以下方式估算：

```text
直连上传 = max(0, 系统上传 - V2Ray 上传)
直连下载 = max(0, 系统下载 - V2Ray 下载)
```

V2Ray 不知道未经过自身的流量，因此插件不会尝试从 V2Ray 查询“直连流量”。系统接口计数包含链路和协议开销，V2Ray 是应用层计数，两者口径并不完全相同，直连结果只能视为近似值。

## v2rayN 配置

v2rayN 的 Xray Core 会根据本地混合端口创建 Metrics 端口：

```text
Xray Metrics 端口 = 本地混合端口 + 4
```

你的当前配置是本地混合端口 `21189`，所以 Metrics 地址为：

```text
http://127.0.0.1:21193/debug/vars
```

在 v2rayN 中开启“显示实时速度/流量统计”并重启 Core。可用以下命令检查接口：

```powershell
(Invoke-RestMethod http://127.0.0.1:21193/debug/vars).stats.outbound | ConvertTo-Json -Depth 8
```

Metrics 仅允许配置 loopback 地址（`127.0.0.1`、`::1` 或 `localhost`），不要将它暴露到局域网或公网。

也可以在项目根目录直接连续采样：

```powershell
python check_v2ray_stats.py --port 21193 --count 10
```

脚本能成功输出累计字节即表示 v2rayN 指标接口可用；采样期间产生代理流量后，还会输出实时速度变化。

## 桌面悬浮窗

- “显示桌面悬浮窗”开关会立即保存并显示或隐藏，不需要再点击保存按钮。
- 悬浮窗只显示代理和直连两组上传、下载速度，不显示系统总流量或其他操作控件。
- 默认背景不透明度为 `0%`，窗口背景完全透明；背景颜色、背景不透明度和字体颜色可在详情页调整。
- 可按住悬浮窗任意区域拖动，松开后自动保存位置。
- 开启“锁定悬浮窗”后禁止拖动并启用鼠标穿透；从网络监控详情页关闭锁定即可恢复拖动。
- 应用启动时插件会直接恢复已保存的显示、锁定、外观和位置状态，无需先打开网络监控详情。
- 插件不再向侧边栏插入额外的“实时网络”摘要按钮，只保留正常的“网络监控”导航入口。
